Review this text using the framework below. Adapt your feedback depth to the content — a 2-line Slack message needs less analysis than a company-wide announcement.

## Audience & Tone Presets

First, identify (or ask) the audience. Apply the matching tone:

| Audience                     | Tone                                | Style                                                            |
| ---------------------------- | ----------------------------------- | ---------------------------------------------------------------- |
| **Executive / leadership**   | Confident, concise, data-driven     | Lead with the outcome, not the process. Numbers > adjectives.    |
| **All-staff / company-wide** | Warm, clear, inclusive              | No jargon. Short sentences. End with a clear action or takeaway. |
| **Client / external**        | Professional, polished, trustworthy | Formal but not stiff. Avoid internal shorthand.                  |
| **Technical / engineering**  | Direct, precise, structured         | Specifics matter. Use bullet points. Skip the fluff.             |
| **Casual / internal team**   | Friendly, brief, natural            | Conversational. Can use contractions and informal phrasing.      |

## Review Checklist

1. **Clarity**: Is the message immediately obvious? Could someone skim it and get the point?
2. **Tone match**: Does it fit the audience above? Flag any tonal mismatches.
3. **Conciseness**: What can be cut without losing meaning? Be aggressive — every word should earn its place.
4. **Impact**: Does it drive the desired action? Is the call-to-action clear?
5. **Flow**: Does the order make sense? Should anything be reordered for better reading?
6. **Format fit**: Is this the right format for the medium? (email vs flyer vs slide vs doc)

## Output Format

- **Audience**: [detected or asked]
- **Issues found**: List specific problems (quote the text, explain WHY it's a problem)
- **Revised version**: Complete rewrite with changes applied
- **Key changes**: Brief notes on what changed and why

If the text is already strong, say so — don't invent problems. If it needs a full rewrite, do it without hedging.
